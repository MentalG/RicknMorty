# RicknMorty

React app built around the Rick and Morty API with routing, filtering, reusable UI primitives, and Redux-powered data flow. Compared to a typical API exercise, this project has a broader app structure with multiple pages and a small watch-list feature stored in local storage.

## Features

- Separate pages for characters, episodes, locations, and watch list
- Filters for character data such as species, status, and gender
- Redux Toolkit state management for remote data
- Reusable UI kit components for inputs, filters, lists, tables, popups, pagination, and loaders
- Local storage powered watch list for tracking episodes

## Stack

- React 17
- Redux Toolkit and React Redux
- React Router
- Material UI
- SCSS
- Rick and Morty API

## Run locally

```bash
npm install
npm start
```

Open `http://localhost:3000` after the dev server starts.

## What this repo demonstrates

- Routing and page-level composition in React
- Shared UI building blocks across multiple views
- Combining remote API data, filters, and local persistence in one app
